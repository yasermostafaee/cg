**Workspace tool choices:**

- **pnpm** for installs (workspace protocol; deterministic).
- **Turborepo** for task orchestration (build, lint, test, package).
- **changesets** for versioning each package independently.
- **TypeScript project references** for fast incremental builds.
- All packages are **TypeScript first**, dual-published as ESM only (Electron 28+ supports ESM).

**Process tier rules** (enforced by ESLint, see В§10):

- `src/main/**` may import Node, `electron`, `fs`, `net`, `better-sqlite3`, `@cg/*`.
- `src/preload/**` may import `electron` only, plus `@cg/shared-schema`, `@cg/shared-ipc`.
- `src/renderer/**` may **not** import Node modules, `electron`, or `src/main/*`. Only `@cg/shared-*`, `@cg/shared-ui`, React, Zustand, etc.
- `src/shared/**` is process-agnostic (no DOM, no Node-only APIs).

## 4. `packages/` вҖ” shared libraries

Each is independently versioned, independently published (private), and follows the same shape:

```
packages/<name>/
в”ңв”Җв”Җ package.json                # name: @cg/<name>; main/module/types fields
в”ңв”Җв”Җ tsconfig.json
в”ңв”Җв”Җ src/
в”Ӯ   в”ңв”Җв”Җ index.ts                # PUBLIC SURFACE вҖ” only what's exported
в”Ӯ   в””в”Җв”Җ ...                     # internal modules, not re-exported
в””в”Җв”Җ tests/
    в””в”Җв”Җ unit/
```

### 4.1 `@cg/shared-schema`

Zod schemas + inferred TS types. **The single source of truth for domain types.**

**Process tier:** runs anywhere (Node, browser, broadcast template).
**Public surface:** every schema is exported; every inferred type uses `z.infer`.
**Forbidden imports:** none. This package has no dependencies except `zod`.

### 4.2 `@cg/shared-ipc`

Typed channel contracts between Electron Main and Renderer. Consumed by both apps.

**Process tier:** Main + Preload + Renderer.
**Public surface:** channel names + Zod schemas + invoke/handle helpers.

### 4.3 `@cg/shared-ui`

shadcn-derived components used by both apps. **No app-specific feature logic.**

**Process tier:** Renderer only.
**Public surface:** components, hooks, theme preset.
**Forbidden imports:** `electron`, Node built-ins, `@cg/caspar-client`, `@cg/persistence`.

### 4.4 `@cg/caspar-client`

The CasparCG runtime client. Implements Phase 5 in full.

**Process tier:** Main only (raw sockets).
**Public surface:** `CasparClient`, event types, config types.
**Forbidden imports:** `electron`, anything renderer-side, `@cg/shared-ui`.

### 4.5 `@cg/template-runtime`

The JS that ships **inside every exported `index.html`**.

**Process tier:** **Broadcast tier** вҖ” runs inside CasparCG's CEF _and_ inside the Designer's preview iframe.
**Public surface:** the runtime bundle (CSS + JS) that the export pipeline copies into `.vcg/index.html`.
**Forbidden imports:** `electron`, Node, `@cg/persistence`, `@cg/caspar-client`, `@cg/audit`.

### 4.6 `@cg/vcg-format`

Pack / unpack / integrity / signing.

**Process tier:** Main only (filesystem, crypto).
**Public surface:** `pack(scene, opts)`, `unpack(path, opts)`, `verify(path)`.

### 4.7 `@cg/lottie-bridge`

Bridges Lottie JSON imports into the scene graph.

**Process tier:** Main (import path) + Broadcast (runtime path). Two entry points: `@cg/lottie-bridge/import`, `@cg/lottie-bridge/runtime`.

### 4.8 `@cg/text-shaping`

RTL/bidi utilities, ZWNJ helpers, Persian digit conversion.

**Process tier:** runs anywhere. Pure functions.

### 4.9 `@cg/telemetry`

Opt-in/opt-out crash reporting and metrics.

**Process tier:** Main only.

### 4.10 `@cg/audit`

Append-only audit log writer.

**Process tier:** Main only.

### 4.11 `@cg/persistence`

SQLite (`better-sqlite3`) wrapped helpers. WAL mode, migrations, prepared statements.

**Process tier:** Main only.

### 4.12 `@cg/eslint-config`

Shared ESLint config including the **import-tier rules** that prevent renderer code from importing Main-only packages.

## 5. `tools/`

Not shipped to users. Build, test, and dev support.

- **amcp-mock** вҖ” standalone Node service used by integration + e2e (satisfies Phase 1 В§7.4)
- **soak-runner** вҖ” 24h harness with memory/queue/OSC drop monitors
- **perf-bench** вҖ” micro-benchmarks for caspar-client, runtime
- **template-fixtures** вҖ” CLI to generate the .vcg files in fixtures/templates/
- **repo-scripts** вҖ” maintenance: changeset orchestrator, version bump, etc.

---

## 6. `fixtures/`

Canonical test data, version-controlled, **not** in any built artifact.

- **templates/** вҖ” reference .vcg packages (lower-third-persian, ticker-bilingual, etc.)
- **osc-traces/** вҖ” NDJSON of recorded bundles (normal-take, failover, split-brain)
- **amcp-sessions/** вҖ” paired req/resp NDJSON (happy-path, 500-recoverable, 502-backoff)
- **fonts/** вҖ” rendering reference cards (persian-shaping, ZWNJ edge cases)

---

## 7. `docs/`

Architecture documents (this series, frozen on approval). Operator/designer end-user docs live here too because they're generated and shipped inside the installers.

- **phases/** вҖ” phase 1вҖ“8 architecture documents
- **adrs/** вҖ” Architecture Decision Records, numbered, never renumbered
- **operator-guide/** вҖ” for the Runtime user (M10 deliverable)
- **designer-guide/** вҖ” for the Designer user (M10 deliverable)

### Forbidden edges (enforced by `@cg/eslint-config/tiers`)

| Source                    | Target                                                                          | Reason                                 |
| ------------------------- | ------------------------------------------------------------------------------- | -------------------------------------- |
| `@cg/shared-ui/**`        | any Main-tier package                                                           | Renderer cannot import Node-only code. |
| `@cg/template-runtime/**` | `@cg/persistence`, `@cg/caspar-client`, `@cg/audit`, `electron`, Node built-ins | Broadcast HTML must be hermetic.       |
| `@cg/caspar-client/**`    | `@cg/shared-ui`, `react`, DOM globals                                           | Main-only library.                     |
| `@cg/shared-schema/**`    | any other `@cg/*`                                                               | Schema must be a sink, not a source.   |
| Any package               | `apps/*`                                                                        | Apps consume packages, never reverse.  |
| `apps/designer/**`        | `apps/runtime/**` (or vice versa)                                               | Apps are siblings.                     |
| `apps/*/renderer/**`      | `apps/*/main/**`                                                                | Process tier boundary.                 |

ESLint catches these at lint time. CI fails the PR.

## 9. Test Layout

Every package and app has `tests/unit`. Apps have `tests/integration` and `tests/e2e`. The runtime app has `tests/soak`.

**Coverage policy:**

- `@cg/shared-schema`, `@cg/vcg-format`, `@cg/caspar-client`, `@cg/template-runtime`: вүҘ 90% statement coverage.
- `@cg/shared-ui`, app `features/*`: visual regression via Playwright screenshots; no statement coverage target.
- `tools/*`: no coverage target; used at dev/CI time.

## 10. Conventions

### Naming

- **Files:** `kebab-case.ts`. Exception: React components are `PascalCase.tsx`.
- **Tests:** `<file>.test.ts` next to source in unit; under `tests/` for integration/e2e.
- **Packages:** `@cg/<kebab-case>`.
- **Channels:** `<area>.<verb>` (e.g., `stack.load`, `templates.list`).
- **Branches:** `phase-7/folder-structure`, `fix/reconciler-resync`, `feat/lottie-import`.
- **ADRs:** numbered, never renumbered. Status: Proposed / Accepted / Superseded.

### Import paths

- `@cg/*` for cross-package imports.
- Relative imports only within a single package.
- No deep imports across packages вҖ” every package's surface is `index.ts`. Enforced by ESLint `no-restricted-imports`.

### TypeScript

- `tsconfig.base.json`: `strict: true`, `noUncheckedIndexedAccess: true`, `exactOptionalPropertyTypes: true`, `noImplicitOverride: true`. No `any`. No `@ts-ignore` without a paired comment justifying it.
- `tsconfig.json` at root: `references` field pointing at every package's `tsconfig.json`. Builds are incremental.

### Versioning

- Apps version together (`@cg/designer` and `@cg/runtime` stay in lockstep major.minor).
- Packages version independently via changesets.
- The `.vcg` `formatVersion` is independent of both вҖ” bumped only when the wire format changes.

## 11. What's intentionally **not** in the repo

| Thing                                           | Where it lives instead                                           |
| ----------------------------------------------- | ---------------------------------------------------------------- |
| Station-specific configs (server IPs, channels) | `%PROGRAMDATA%\BroadcastCG\config\station.yaml` (per-deployment) |
| Signing keys for `.vcg`                         | Out of repo; passed to CI via secrets                            |
| Code-signing EV cert                            | Out of repo; passed to release CI                                |
| Telemetry endpoint URL                          | Compile-time constant, injected by release CI                    |
| Operator-authored `.vcg` templates              | Stations' own template libraries                                 |
| CasparCG itself                                 | Stations' own playout servers                                    |
| Fonts beyond the bundled set                    | Station-specific; loaded from `%PROGRAMDATA%` at runtime         |
