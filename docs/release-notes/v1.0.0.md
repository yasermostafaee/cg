# v1.0.0 — Release Notes

First production release of the Broadcast CG Platform. A pair of Windows desktop apps — **Designer** for building broadcast templates and **Runtime** for taking them on air against CasparCG 2.3.x — plus their shared schema and template-runtime libraries.

## Highlights

- **Persian/RTL is native.** Bidi-aware text editing in Designer with ZWNJ insertion, Persian digit/Jalali date formatters at the runtime, and a hand-tuned reference template the QA suite exercises every commit.
- **Two CasparCG servers, one operator surface.** `mirror-sync`, `mirror-async`, and `journal-replay` strategies in `@cg/caspar-client`. Auto-failover on ping miss / OSC silence / command-timeout burst / 5xx burst. Manual failover from the Runtime status bar. Split-brain detection with corrective resend.
- **Deterministic `.vcg` packaging.** Re-exporting the same scene produces a byte-identical archive. Merkle-hash integrity; `verify()` catches a single-byte flip before the watched folder hands the file to the runtime.
- **Air-critical contracts.** No installs while anything is on-air (`UpdateGate`). Lock mode prevents accidental TAKEs. Append-only NDJSON audit log survives restart. Telemetry off by default — zero outbound network requests until the operator opts in.

## Designer

- Library with **6 starter templates** covering every Phase 3 §5 type (logo-bug, lower-third × 2, ticker, breaking-news, fullscreen) — Persian Reference Render is the load-bearing QA template.
- Canvas with text / shape / image element creation. Inline text editing with RTL-aware caret and Shift+Space ZWNJ.
- Inspector: Transform / Style / Animation / Bindings sections.
- Animation Inspector covers all Phase 3 §5 preset kinds — 5 entry × 4 loop × 5 exit = **100 combinations**, every one exercised by the M7.4 fixture test.
- Dynamic fields panel with **bind-from-canvas** workflow. Resolver picks the most natural binding for each (field type, element type) pair.
- Bottom dock surfaces both the **animation timeline strip** and the **live Issues panel**. Preflight blocks export on `error`-severity issues (unbound required field, unknown binding target, missing asset).
- Deterministic export pipeline producing `.vcg` archives with bundled `@cg/template-runtime` + manifest + assets + font references.

## Runtime

- Watched-folder ingest verifies each `.vcg` before registering its template id.
- Stack panel with load → take → out → remove flow.
- **Editable Inspector** — operator field edits commit-on-blur and fire a live AMCP `CG INVOKE update` per ADR 0006.
- **Failover banner** + manual `⇄ FAILOVER` button. Banner re-asserts on each new failover even if the previous one was dismissed.
- **Audit panel** tails the NDJSON log with action/actor filters.
- **Lock mode** with PIN engage/release, elapsed-time chip, wrong-PIN counter, and audit entries on every state change.
- **Telemetry mode**: `off` (default) / `on` / `air-gapped`. `isOutboundAllowed()` gates anywhere considering a `fetch()`.
- **Update gate** refuses electron-updater installs while any stack item is playing / on-air / updating / exiting.

## Known limitations

- **`cg.update` payload not auto-delivered.** CG INVOKE returns 202 but the JSON arg doesn't reach `window.update` (ADR 0006). The runtime _issues_ the call but operators wanting verified payload delivery should re-take the row. Tracked for v1.1.
- **24 h soak + on-site pilot.** Code-level perf budgets enforced in CI; the 24 h hardware soak and pilot-station sign-off are infrastructure-dependent.
- **EV code-signed installers.** Windows code-signing pipeline + EV cert procurement land alongside the public release infrastructure. Installer config skeletons (electron-builder) ship in this repo; the EV signing key is operator-supplied.

## Upgrade notes

- First v1; no migration path. Schema version `1` is baked in.
- Settings file location: `$CG_SETTINGS_FILE`, defaults to a per-OS tmp path. Audit log location: `$CG_AUDIT_FILE`.
